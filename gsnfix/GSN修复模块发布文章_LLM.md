这是针对TB710FU，联想小新Pad Pro GT,Lenovo Xiaoxin Pad Pro GT的知识库文档。

刚刚救砖成功，却发现使用触控笔时提示未写入GSN？在线壁纸莫名提示网络问题？

这都是属性`ro.odm.lenovo.gsn`导致的问题。

正常情况下这个属性应当为你的序列号，例如`HA1145VM`（开头两个字母一定为HA，后面跟随6位由数字和大写字母混合的随机字符串），但当使用救砖包刷写或者从第三方ROM刷写回ZUXOS之后，这个属性会被覆盖。

要验证是否是这个属性导致的问题，在拥有root权限的终端模拟器上输入以下命令：

```bash
su -c getprop ro.odm.lenovo.gsn
```

如果是这个属性引发的问题，屏幕上应当出现一个空行

这个时候可以试试向这个属性写入你的序列号，例如下面这样

```bash
su -c setprop ro.odm.lenovo.gsn HA1145VM
```

HA1145VM是上文瞎编的序列号，使用时应当替换成你自己的序列号

执行命令后应当没有任何提示，终端等待下一条命令被输入

成功执行setprop之后就可以试试在线壁纸等功能是否恢复正常了，如果仍旧提示网络问题可以再试几次setprop，但事不过三，如果三次还不行的话说明不是这个问题。

如果不确定序列号是多少，可以执行下面的命令：

```bash
su -c getprop ro.serialno
```

这样应当能获取到SN码。在设置、机身贴纸或其他地方查询也可以。

不过这样的修复方式在重启后就失效了。现在为了方便，编写一个KernelSU模块（Magisk未测试，应该也是可用的），在每次开机时自动获取序列号并执行setprop命令。

模块可以在这个链接找到： https://wwcp.lanzout.com/iZaoq3awznfe
