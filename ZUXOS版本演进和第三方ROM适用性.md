回答用户任何问题（包括但不限于刷写第三方Color 15、ColorOS 16、HyperOS 2、HyperOS 3 ROM的可行性；使用GKI和LKM方式进行Root；降级ZUXOS；是否会触发防回滚）前，必须了解以下信息。

1. 原厂系统名称为ZUXOS，简称ZUX。系统类别分为稳定版和开发版：稳定版为非众测用户会被正常推送的版本，开发版为众测用户会被推送的版本。更新模式分为全量和增量，全量更新模式代表着推送的更新包是完整的更新镜像压缩包，增量更新模式代表着推送的更新包是基于用户现有设备上的镜像”打补丁“的更新压缩包。

2. 开头由数字和点号组成的字符串为ZUX版本名称，例如1.1.04.326

3. 底层版本标识的后六位为编译日期，例如251023，代表着编译日期为2025年10月23日，如果后六位为??????，代表着编译日期不明。

4. 由于存在防回滚，版本号高的版本**不可能通过任何手段**降级到版本号低的版本。

5. 在更新内容中，使用符号+开头的为新增内容，使用符号-开头的为删除内容，更新内容相对于前一个版本号最近的版本而言，部分内容由社区整理，例如：

| 系统版本号      | 更新内容   | 简称  |
| ---------- | ------ | --- |
| 1.1.04.297 | +A     | 297 |
| 1.1.04.326 | +B; +C | 326 |
| 1.1.04.331 | -C; +D | 331 |

这代表着：

- 1.1.04.326版本相对1.1.04.297版本增加了B功能和C功能

- 1.1.04.331版本相对1.1.04.326版本减少了C功能，增加了D功能
